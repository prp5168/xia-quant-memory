## 观察模式（2026-03-21 鹏哥决策，当前状态）

### 为什么进入观察模式
- 43笔已平仓，胜率14%，累计PnL -175u，模型信号方向判断大面积失准
- 核心问题：不知道模型算出的 edge 是否真的有预测价值
- 决策：暂停一切建仓，纯观察，先验证再恢复

### 观察模式下必须记录的内容
1. **observe-log.jsonl - 概率路径观察（每轮扫描自动写入）**
   - 每个城市/日期标的的 TWC 预报 max、调整后 mu、sigma
   - **所有** market bins 的 model概率 vs 盘口概率 vs edge（不只是 ±2°C）
   - 是否是当天标的、当地几点
   - 结算日当天 local 16:00 后停止记录（最高温已实现）
   - 目的：追踪"模型说有edge → 后续edge是否收敛 → 结算结果"的完整路径

2. **forecast-log.jsonl - TWC预报误差积累（每轮扫描自动写入）**
   - forecastMax、hourlyMax、mu、sigma、天气参数（风/降水/云/湿度）
   - 提前天数（dayDiff）
   - 目的：积累预报误差样本，验证 sigma 参数是否合理

3. **watchlist.json - 城市追踪延续（每轮扫描自动维护）**
   - volume top10 城市自动写入
   - 有 edge 信号的城市也写入
   - 城市不因 volume 排名波动而丢失观察，直到结算日过期自动清理
   - 目的：解决"昨天发现好 edge 今天 volume 掉了就不看了"的问题

4. **双日志闭环要求（2026-03-22 新增）**
   - 纳入 watchlist / 扫描名单的城市，只要该 city+date 仍处于有效记录窗口内，`observe-log.jsonl` 和 `forecast-log.jsonl` 都应写入，不能只写一边
   - 统一记录窗口：城市当地时间当天 16:00 后，最高温基本兑现，observe/forecast 两边都停止写入
   - 若该 city+date 当轮没有取到 PM 市场，observe-log 也要写一条占位记录（`marketMissing=true`, `bins={}`），这样后续核查能分清是"没记录"还是"当轮没盘口/抓取失败"
   - 目的：保证后续做 edge 收敛验证时，盘口概率路径和预报路径能严格对齐

### 观察期要验证的三个核心问题
1. **edge 收敛验证**：模型算出的 edge，后续盘口是否向模型方向收敛？（用 observe-log 追踪同一标的多轮概率变化）
2. **BUY_YES 方向问题**：历史上 BUY_YES 几乎全错，为什么？是模型系统性偏差还是市场定价逻辑不同？
3. **模型 vs 市场维度差异**：市场对精确温度档的定价是否和正态分布+sigma模型在不同维度？

### 观察期新增诊断框架（2026-03-22 晚确认）
- 鹏哥认可一个更清晰的诊断思路：**把系统拆成结算层 / 概率层 / 交易层三层分别看**，不要再把亏损直接混同为"预测错"或"市场不可做"。
- 当前最该吸收的优化方向：
  1. **历史 forecast snapshot 是最大数据缺口**：现有 observe-log / forecast-log 只能从现在开始积累，过去每个决策时点的 forecast 版本缺失，会限制严格回测；后续需要重点补 forecast archive / snapshot 数据层。
  2. **时间窗切片分析**：后续所有统计至少按 dayDiff、localHour、本地早晨/中午/下午前窗口拆开，不把全天样本混算。
  3. **概率校准而非只看方向**：后续样本够了，要验证模型给出的 30%/50%/70% 是否真的对应历史命中率，识别系统性过度自信/过度保守。
  4. **先继续观察，不急着大改交易规则**：现阶段先补数据层、继续记录、继续观察，不因为一段分析就立刻重构执行逻辑。
- 当前阶段的核心结论：**先把 forecast snapshot 数据层做厚，再按"时间窗 + 概率校准 + 收敛路径"去判断问题到底在信号还是执行。**

## 潜在未来方向：同档位点差交易 / 被动做市（2026-03-23 讨论记录）
- 鹏哥提出一个未来实盘研究方向：不是跨档位组合套利，而是**同一城市、同一温度档位内部**，在买一/卖一价差很大时，结合轻方向判断去做被动挂单，承担提供流动性的角色，在 bid 附近接货、在 ask 附近卖出，赚点差与短时重估。
- 讨论结论：
  - 这不是严格无风险套利，更准确是**带库存风险的被动做市 / 半做市型点差交易**。
  - 这条路值得研究，可能比单腿裸押结算更接近交易正路，但当前只是研究方向，尚未验证可实盘。
  - 最大风险不是温度本身，而是库存风险：接到货后卖不出去，或者方向错了继续被动吃亏。
- 后续若继续研究，评估重点应放在：
  1. 是否有真实成交流，而不是只有名义大 spread
  2. 成交后能否快速卸掉库存
  3. spread 收益能否覆盖错误方向时的损失
  4. 这种被动接 bid 再挂 ask 的方式，是否长期优于主动追价
- 目前状态：**先记录，不立刻展开策略设计；等需要时再继续分析。**
