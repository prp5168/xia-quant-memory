## 汇总口径 bug 修复（2026-03-27 早）
- 鹏哥指出 cron 推送中的汇总数字异常：同一条报告里 `持仓5个 / 已平仓11笔` 对得上，但 `现金$288.80 / 累计PnL -13.34` 与 `data/portfolio.json` 原始账面不一致。
- 排查结果：`scripts/sim_portfolio_scan.mjs` 的 summary 仍在使用一套旧的 BUY_NO “纠偏汇总口径”（`getCorrectedSummaryAccounting` / `getCorrectedClosedTradePnl`）。
- 这套纠偏逻辑原本是为了修复过去把 BUY_NO 当成 `1-YES` 处理时的历史汇总误差；但在 BUY_NO 已全面改成 **direct PM NO price** 之后，这套旧纠偏继续存在，反而把 summary 里的 cash / cumulative PnL 算错。
- 已修复：summary 改为直接使用真实账面字段：
  - `cash = pf.cash`
  - `累计PnL = pf.totalPnl`
  - `总资产 = pf.cash + 当前订单簿可平资产`
- 修复后实跑校验通过，最新输出：`持仓4个｜现金$326.54｜可平资产$67.47｜总资产$394.01｜已平仓12笔｜累计PnL $4.58`。
- 结论：以后 Telegram 推送汇总以 `portfolio.json` 实时账面为准，不再走旧纠偏汇总函数。
