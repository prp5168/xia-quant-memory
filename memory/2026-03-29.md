
## 11:43 GMT+8：动态仓位上线（commit 88540a8）

### 变更内容
- 建仓金额从"固定上限"改为**基于盘口深度+edge的动态调整**
- 规则上限仍为 `rules.maxSingleTrade`（主盘$10 / YES实验$5），但实际下单金额会根据盘口状况自动缩减

### 动态仓位逻辑
1. **根据 edge 确定最大可接受滑点**：
   - edge ≥ 30%: 容忍 10% 绝对滑点
   - edge 20-30%: 容忍 5%
   - edge 15-20%: 只容忍 3%
2. **在可接受滑点范围内累加 ask 档位总容量**（不是只看第一档）
3. **实际预算 = min(上限, 现金, 可接受滑点内深度)**
   - 深度够 → 按上限下
   - 深度不够 → 只下深度允许的量，不硬推价格
4. **滑点二次检查**：如果实际吃单后滑点 > edge 的一半，缩减到只吃第一档

### 为什么要看"可接受滑点范围内的总容量"而不是"第一档容量"
- 尾部低价 NO（每股 1-2%）第一档 $ 容量可能只有 $0.06，但后续 20 档都在 1-2% 范围内
- 如果只看第一档 $ 容量，尾部标的永远只能下 $2（最低值），太保守
- 按"滑点范围"累加才能正确反映实际可用深度

### 建仓日志新增
- 每笔建仓会输出：`📏 仓位: 上限$X | 滑点容忍Y% | 深度内$Z → 预算$W`
- 便于后续验证动态仓位是否合理
