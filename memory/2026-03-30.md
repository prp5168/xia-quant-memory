### 手续费计算规则
- fee = base_fee_rate × min(price, 1-price) × shares
- 持有到结算（赢）：只收入场 fee，结算兑付无额外 fee
- 中途卖出：入场 + 卖出各收一次 fee
- **对 BUY_NO 策略：fee 固定吃掉赢利的 10%**（因为 min(noP, 1-noP) = 1-noP for NO≥50%，而 (1-noP)/shares = 赢利率）

## 08:26 GMT+8：发现严重 BUG — °C/°F 桶位混合计算

### BUG 描述
- PM 美国城市盘口用 °F（k=54 表示 54-55°F），非美国城市用 °C
- 但模型 mu/sigma 始终用 °C（来自 TWC API metric 单位）
- dist 计算用 `|k - mu|`，没有单位转换
- 导致所有美国城市的 dist 虚高 30-50 → **全部被标成 tail**
- 例：Seattle k=54°F, mu=12.7°C(≈55°F), dist=41.3 → 标为 tail
- 实际 |54°F - 55°F| = 1°F → 应该是 **center**

### 影响范围
- **65 笔°F城市交易中，34 笔 tail→center（最危险），23 笔 tail→near**
- 只有 **8 笔桶位正确**（12%）
- 受影响城市：Miami, NYC, Chicago, Atlanta, Dallas, Seattle, LA, SF
- **当前 15 笔持仓被错标为 tail 但实际是 center，总成本 $232**

### Seattle 03-28 k=54 具体分析
- TWC 实测：Seattle 03-28 最高温 = **55°F = 13°C**
- PM 结算：54-55°F 档命中 → YES resolve → **NO 归零**
- 我们 BUY_NO k=54 → **全亏 -$19.52**
- 入场时 mu=12.7°C ≈ 55°F，k=54°F，实际只差 1°F = center
- 系统误判为 tail (dist=41.3) → 适用了 tail 持有到结算规则 → 没止损直接结算归零
- **如果桶位正确标为 center，按 V2.1 规则根本不该建仓**

### 幸存原因
- 03-29 的°F城市虽然桶位全错，但大多数实测温度偏离更远（如 Dallas actual=52°F vs k=80°F），所以 NO 还是赢了
- 但这只是运气，不是策略正确

### 需要修复
- dist 计算必须统一单位：要么 mu 转°F 再算 dist，要么 k 转°C 再算 dist
- bucket 阈值也需要调整：°F 每档 2°F，°C 每档 1°C，center/near/tail 的距离阈值应不同
- 这个 BUG 同时影响建仓判断、止盈止损路径选择、回测统计
