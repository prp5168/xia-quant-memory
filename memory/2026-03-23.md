## 点差套利 / 被动做市方向讨论（2026-03-23 早）
- 鹏哥提出一个重要的未来实盘思路，不是跨档位组合套利，而是**同一城市、同一温度档位内部的点差交易**：
  - 当某个档位方向判断偏看涨（或偏看跌）
  - 且同一档位买一/卖一价差很大（有时十几个点）
  - 通过承担提供流动性的角色，在买一附近接货，再挂到卖一附近卖出，赚点差 + 短时重估收益
- 讨论后的核心结论：
  1. 这不是严格无风险套利，更准确说是**带库存风险的被动做市 / 半做市型点差交易**。
  2. 它赚的不是最终结算本身，而更像赚三类收益：
     - spread 本身
     - 短时方向优势（轻方向判断）
     - 给薄盘口补流动性的报酬
  3. 这条路**值得研究**，而且可能比单腿裸押结算更像交易正路；但现在还只是研究方向，不能直接当成已验证可实盘策略。
  4. 最大风险不是温度预测错，而是**库存风险**：买到了，但后面没人接你的卖单，或者方向错了成了接飞刀的人。
  5. 主要风险被明确拆成 5 类：
     - 方向对了但流动性不来，仓位卡死
     - 方向错了，接到继续下跌的库存
     - 名义 spread 很大，但真实可成交 spread 很小
     - 单档价格受整条分布重排影响，不是独立资产
     - 越接近结算，信息不对称越强，补流动性不一定占优
  6. 若后续研究该方向，只有在以下条件同时满足时才值得考虑：
     - 对方向有轻度优势（不是纯瞎挂）
     - spread 足够厚，能覆盖错误方向和退出损耗
     - 有真实成交流，不是死盘口
     - 时间窗合适（有波动、有流动性、但未进入最后踩踏阶段）
  7. 后续更准确的内部命名：**被动点差捕捉策略 / 带方向过滤的被动做市**，不要草率称为“无风险套利”。
  8. 若未来继续分析，这个策略需要单独定义日志与评估框架，至少记录：
     - 城市 / 日期 / 温度档
     - bid / ask / spread
     - 当时 fair 与方向判断
     - 是否能在 bid 成交
     - 成交后多久能在 ask 附近卖出
     - 若卖不出去，最大浮亏和最终退出价格
  9. 未来判断该模式是否成立，不看单纯胜率，而看：
     - 成交后能否快速卸库存
     - spread 收益能否覆盖少数错误方向损失
     - 是否比主动追价买入更优
