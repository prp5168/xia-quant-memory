# Quant Sync Policy

## 默认策略
- 日终自动同步：每天 23:55（Asia/Shanghai）执行一次。
- 这是保底同步，不是唯一同步。

## 必须即时同步的情形
出现以下任一类变更时，必须在**当轮写入文件后**立即执行一次 GitHub 同步：

1. 逻辑修改
   - 扫描逻辑、edge 计算、概率模型、止盈止损逻辑变化
2. 脚本修改
   - `scripts/` 下任何影响功能或数据口径的改动
3. 数据/配置修改
   - `portfolio.json`、`watchlist.json`、关键参数、配置文件人工变更
4. 方向修改
   - 如交易 → 观察、观察 → 恢复交易、研究重点切换
5. 关键结论确认
   - 鹏哥明确拍板的重要策略结论、风险结论、方法论结论
6. 重大错误复盘
   - 发现会影响后续判断或执行纪律的错误并完成修正

## 执行命令
```bash
cd /root/.openclaw/workspace/agents/weather-odds-bot
./scripts/auto_sync.sh --reason "logic update"
```

## 原则
- 数据日志的常规增长不要求每轮都推。
- 关键变更必须即时推；普通数据增量由日终同步兜底。
- 先写文件，再同步 GitHub；没有写入文件的“口头记住”不算数。
